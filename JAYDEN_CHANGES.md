# Jayden Changes

Summary of the changes made on the `jayden_branch` (uncommitted working-tree changes as of 2026-06-20).

The overarching theme: **the trace storage model was redesigned from a normalized 3-table schema into a single flat `CALLS` table**, and the backend, SDK, and tests were all updated to match. A `project_id` concept was also introduced end-to-end, plus seed data for local testing.

---

## 1. Database schema redesign — `supabase/migrations/001_calls.sql`

Replaced the normalized schema (`traces` + `inputs` + `outputs`) with a flat design.

**Before:** three tables joined by `trace_id`
- `traces` — metrics (UUID PK, `gen_random_uuid()`)
- `inputs` — `user_input`, `system_prompt`, `full_prompt`
- `outputs` — `code`

**After:** two tables
- **`PROFILES`** (new) — `id` (BIGINT identity), `email`, `API_KEY`, `created_at`
- **`CALLS`** — one row per traced LLM call, holding everything inline

Key changes in `CALLS`:
- PK switched from `UUID` to `BIGINT GENERATED BY DEFAULT AS IDENTITY`
- Field renames:
  - `cost_usd` → `cost` (now `REAL`)
  - `status` (text, default `'ok'`) → `status_success` (`BOOLEAN`)
  - `code` → `output_code`
- Added `project_id BIGINT`
- Dropped columns: `provider`, `system_prompt`, `full_prompt`, `context_limit`, `context_utilization`, `finish_reason`
- Token counts widened to `BIGINT` with defaults (`output_tokens`/`reasoning_tokens`/`total_tokens` default `0`)
- Indexes renamed: `idx_calls_run_id`, `idx_calls_created_at`
- Dropped the `pgcrypto` extension requirement (no longer need `gen_random_uuid()`)

---

## 2. Backend schemas — `backend/schemas/trace.py`

Aligned the Pydantic models with the new `CALLS` columns.

- `IngestPayload`:
  - Removed `provider`, `system_prompt`, `full_prompt`, `context_limit`, `context_utilization`, `finish_reason`
  - `cost_usd` → `cost`, `status` → `status_success` (bool, default `True`), `code` → `output_code`
  - Added `project_id: int | None`
- `TraceRecord`:
  - Mirrors the `CALLS` read shape (adds `created_at`, `status_success`, `run_id`, `project_id`; drops old `status` string)

---

## 3. Backend service — `backend/services/trace_service.py`

- Updated `_CALL_SELECT` column list/order to match `CALLS`
- `_row_to_trace` now maps `status_success`, `created_at`, `run_id`, `project_id` directly (no more `"success"/"error"` string conversion)
- `ingest_trace` INSERT updated: new column list, removed `provider`, added `run_id` + `project_id`, uses `cost`, `status_success`, `output_code`

---

## 4. SDK changes — `sdk/src/`

- **`types.ts`**
  - `TraceConfig` gains `projectId?: number`
  - `TracePayload`: added `project_id?` and `output_code?`; renamed `cost_usd` → `cost`, `status` (`'success' | 'error'`) → `status_success: boolean`; removed `context_limit` / `context_utilization`
- **`tracer.ts`**
  - `Tracer` now stores `projectId` from config and exposes it
- **`wrappers/anthropic.ts`**
  - Threads `projectId` through (`_trace.projectId ?? tracer.projectId`)
  - Added `extractOutputCode()` to pull text content out of the Anthropic response into `output_code`
  - Removed context-window logic (`getContextWindow`, `context_limit`, `context_utilization`)
  - Emits `cost`, `status_success`, `output_code` instead of old fields
  - Minor type tidy: `(response.usage as { thinking_tokens?: number })` instead of `as any`

The compiled `sdk/dist/*` artifacts were rebuilt to match.

---

## 5. Tests

- **`sdk/smoke-test.mjs`** — asserts on `status_success`, `cost`, and `output_code` (includes `is_palindrome`) instead of `status`, `cost_usd`, `context_limit`
- **`sdk/e2e-test.mjs`** — asserts `status_success === true` instead of `status === 'success'`

---

## 6. New files — seed data

- **`seed/calls-test-data.json`** — sample `CALLS` rows (steps like `classify-intent`, `draft-reply`, `proofread`) sharing a `run_id` and `project_id`, used for local testing without live LLM calls
- **`backend/seed_calls.py`** — loads the JSON above into Supabase via `IngestPayload` + `ingest_trace`, no server required
